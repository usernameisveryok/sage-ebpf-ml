% ABSTRACT

\begin{abstract}
Deploying neural networks inside the Linux kernel via eBPF/XDP enables
microsecond-level network intrusion detection, yet the eBPF execution
environment forbids floating point, limits the stack to 512 bytes, and
caps instruction budgets---making integer multiplication the dominant
performance bottleneck.
We present \sys (\textbf{S}afe \textbf{A}daptive In-Kernel Neural
Inference En\textbf{g}in\textbf{e}), the first eBPF/XDP neural network
that eliminates \emph{all} multiply instructions through two
complementary operators: \textbf{MapLUT}, which replaces the input layer
with BPF-map-based lookup tables, and \textbf{TernaryShift}, which
reduces hidden-layer weights to $\{-\alpha, 0, +\alpha\}$ and replaces
multiplication with bit shifts.
Naively applying ternary quantization collapses minority-class detection
(PortScan F1 drops from 0.597 to 0.013); our two-phase
\textbf{Quantization-Aware Training} with Straight-Through Estimation
and focal loss recovers and even \emph{surpasses} the float baseline
(PortScan F1 = 0.629).
The full MapLUT\,+\,Ternary-QAT pipeline achieves 95.78\% accuracy on
CIC-IDS-2017 with 1{,}424 BPF instructions---none of which are
multiplications.
\end{abstract}

\section{Introduction}\label{sec:intro}
% ============================================================

Network attacks now routinely exceed hundreds of gigabits per second,
demanding detection latencies that user-space processing pipelines
simply cannot meet.
The eBPF/XDP subsystem in the Linux kernel offers a compelling
alternative: programs attached at the eXpress Data Path (XDP) hook
intercept packets at the NIC driver level---before they enter the
kernel network stack---achieving single-digit microsecond
latency~\cite{zhang2024dsn}.
This has motivated a growing body of work on deploying machine learning
classifiers directly inside the kernel for real-time intrusion
detection~\cite{bachl2021dt, hara2023nn, zhang2024dsn, osaki2024,
chen2025kd}.

However, the eBPF verifier enforces constraints that make neural
network deployment non-trivial.
Programs cannot use floating-point arithmetic, are limited to a
512-byte stack per program instance, and face an instruction budget of
one million verified instructions.
More critically, \emph{integer multiplication is the most expensive
eBPF instruction}---requiring multi-cycle emulation on architectures
such as 32-bit ARM where it lacks a hardware primitive---and dominates
both instruction count and latency in existing in-kernel neural
networks~\cite{zhang2024dsn, hara2023nn}.
Prior systems accept this cost as unavoidable; none have attempted
multiply-free inference.

Our key insight is that \emph{zero-multiply neural inference is
achievable through operator co-design}, replacing multiplications with
lookup tables and bit shifts at each layer.
However, naively applying ternary weight quantization to models trained
on imbalanced security data causes catastrophic minority-class collapse:
PortScan F1 plummets from 0.597 to 0.013 (Table~\ref{tab:ablation}).
We show that Quantization-Aware Training (QAT) with the
Straight-Through Estimator (STE)~\cite{bengio2013ste} and focal
loss~\cite{lin2017focal} resolves this problem, and when combined with
lookup-table-based input layers, yields the \emph{first fully
zero-multiply neural network in eBPF/XDP}.

\sys makes four contributions:

\begin{enumerate}[leftmargin=*,itemsep=2pt]
\item \textbf{Lossless int32 fixed-point inference}
(\S\ref{sec:background}).
The enlargement method~\cite{zhang2024dsn} with scale $s = 2^{16}$
achieves identical accuracy to the float baseline: 97.64\% on
CIC-IDS-2017 (578K test samples, 4 classes).

\item \textbf{MapLUT: lookup-table input layer}
(\S\ref{sec:maplut}).
Precomputed partial-sum tables stored in BPF \texttt{PERCPU\_ARRAY}
maps replace all 768 input-layer multiplications with table lookups.
Because the LUT resides in a BPF map, model weights can be hot-updated
at runtime with zero downtime---a capability no prior in-kernel ML
system offers.

\item \textbf{QAT with STE for ternary quantization}
(\S\ref{sec:qat}).
Post-hoc ternarization destroys minority classes (PortScan F1: 0.020;
BruteForce F1: 0.039).
Our two-phase procedure---float warmup with focal loss followed by
STE-based ternary training---recovers minority-class performance
and \emph{exceeds} the float baseline's PortScan~F1
(0.629 vs.\ 0.597).

\item \textbf{MapLUT + Ternary-QAT = zero-multiply NN}
(\S\ref{sec:eval}).
The full pipeline achieves 95.78\% accuracy (512 bins) with 1{,}424 BPF
instructions, zero of which are multiplications
(Table~\ref{tab:headline}).
\end{enumerate}

% ============================================================
\section{Background}\label{sec:background}
% ============================================================

\subsection{eBPF/XDP Execution Model}

Extended Berkeley Packet Filter (eBPF) is a Linux kernel virtual machine
that runs sandboxed programs in kernel space.
Before a program is loaded, the eBPF verifier statically checks it
against a strict set of constraints:
(i)~no floating-point instructions;
(ii)~bounded loops only (Linux~5.3+);
(iii)~a 512-byte stack per program instance;
(iv)~a maximum of $10^6$ verified instructions; and
(v)~no dynamic memory allocation.
eXpress Data Path (XDP) attaches eBPF programs at the NIC driver level,
enabling wire-speed packet processing before the kernel network stack.

Two mechanisms are central to working within these limits.
\textbf{Tail calls} allow one eBPF program to chain to another without
returning, each receiving its own 512-byte stack frame (maximum chain
depth: 33).
This effectively circumvents the stack limitation by distributing
computation across multiple programs.
\textbf{BPF maps} are kernel-managed key--value stores shared between
eBPF programs and user space; per-CPU array maps provide lock-free,
cache-friendly storage with zero contention across cores---a property
we exploit extensively.

\subsection{Enlargement-Method Quantization}\label{sec:enlarge}

The enlargement method~\cite{zhang2024dsn} converts a floating-point MLP
to integer-only arithmetic.
Given a scale factor $s = 2^b$, weights are quantized as
$W_E = \text{round}(s \cdot W)$ and biases as
$B_E = \text{round}(s \cdot B)$, both stored as \texttt{int32}.
For each layer, inference computes:
\begin{equation}\label{eq:enlarge}
z_i = \Bigl(\sum_{j} W_E[i][j] \cdot x[j]\Bigr) \gg b \;+\; B_E[i]
\end{equation}
The accumulation uses \texttt{int64} to prevent intermediate overflow,
and the final argmax is scale-invariant, requiring no de-quantization.
On our CIC-IDS-2017 task with architecture $[12, 64, 64, 4]$, this
method achieves \emph{lossless} quantization: 97.64\% accuracy,
identical to the float baseline (Table~\ref{tab:ablation}).
This forms the starting point for \sys; the subsequent operators replace
the per-layer multiply operations in Eq.~\eqref{eq:enlarge} with
multiply-free alternatives.

\subsection{Prior eBPF ML Systems}

Table~\ref{tab:prior} summarizes prior in-kernel ML systems for network
intrusion detection.
Bachl~\etal~\cite{bachl2021dt} deployed decision trees in eBPF for
CIC-IDS-2017 classification, achieving ${\sim}$99\% binary accuracy but
with a $>$32\,KB memory footprint.
Hara~\etal~\cite{hara2023nn} demonstrated int8 neural inference in
eBPF with 84\% latency reduction over user space, but at the cost of
significant accuracy degradation.
Zhang~\etal~\cite{zhang2024dsn} introduced the enlargement method with
tail-call chains, the most closely related work to ours, but limited to
binary classification and fully reliant on multiply instructions.
Osaki~\etal~\cite{osaki2024} proposed dynamic fixed-point quantization,
and Chen~\etal~\cite{chen2025kd} combined knowledge distillation with
decision trees.
All of these systems require multiply instructions; \sys is the first
to achieve zero-multiply inference.

\section{System Design}\label{sec:design}
% ============================================================

\sys classifies network flows into four categories---BENIGN, DDoS,
PortScan, and BruteForce---using an MLP with architecture
$[12, 64, 64, 4]$: 12 flow features extracted from packet headers,
two 64-unit hidden layers with ReLU activation, and a 4-class softmax
output.
The inference pipeline is partitioned into a 4-stage tail-call chain,
with per-CPU array maps providing lock-free intermediate storage between
stages.

Two distinct training procedures produce two models that serve
complementary roles in the evaluation:

\begin{itemize}[leftmargin=*,itemsep=2pt]
\item \textbf{Original model}: trained with standard cross-entropy loss
(100 epochs, 100K/class oversampling).
This model produces the Float, Int32, and all post-hoc quantization
results.
\item \textbf{QAT model}: trained with the two-phase procedure described
in \S\ref{sec:qat}.
This model is optimized exclusively for ternary deployment and produces
all Ternary-QAT and MapLUT\,+\,Ternary-QAT results.
\end{itemize}

\subsection{MapLUT: Lookup-Table Input Layer}\label{sec:maplut}

\textbf{Problem.}
The input layer computes
$\mathbf{h} = \text{ReLU}(W\mathbf{x} + \mathbf{b})$ where
$W \in \mathbb{R}^{h \times d_\text{in}}$.
For our architecture, this requires
$d_\text{in} \times h = 12 \times 64 = 768$ multiply-accumulate
operations---the single largest source of multiplications in the
network.

\textbf{Insight.}
Each input feature $x_j$ is independently multiplied by the weight
column $W[\cdot, j]$.
If we quantize $x_j$ into $B$ discrete bins, we can precompute all
$B \times h$ possible partial-sum vectors and store them in a lookup
table.
At inference time, a single table lookup replaces $h$ multiplications.

\textbf{Bin quantization.}
Each feature $j$ is mapped to a bin index using only subtraction and
bit shift:
\begin{equation}\label{eq:bin}
\text{bin}_j = \text{clamp}\!\bigl(\,(x_j - \text{offset}_j)
\gg \text{shift}_j,\; 0,\; B{-}1\bigr)
\end{equation}
The per-feature offset and shift parameters are calibrated from
training-data statistics to cover the input distribution uniformly.
Crucially, Eq.~\eqref{eq:bin} contains no multiplication.

\textbf{LUT construction.}
For each feature $j$, bin $b$, and hidden unit $i$, the table entry is:
\begin{equation}\label{eq:lut}
\text{LUT}[j, b, i] = \text{round}\!\bigl(W[i,j] \cdot
\text{center}(j, b) \cdot s\bigr)
\end{equation}
where $\text{center}(j, b)$ is the representative value of bin $b$ for
feature $j$, and $s = 2^{16}$ is the quantization scale.
The full LUT is computed offline and loaded into a BPF
\texttt{PERCPU\_ARRAY} map at program startup.

\textbf{Inference.}
The forward pass for layer~0 reduces to pure accumulation:
\begin{equation}\label{eq:lutinfer}
\text{hidden}[i] \mathrel{+}= \text{LUT}[j, \text{bin}_j, i]
\quad \forall\, j \in [0, d_\text{in})
\end{equation}
This is $d_\text{in}$ additions per hidden unit---zero multiplications.
For $B = 512$ bins and $h = 64$, the LUT occupies
$12 \times 512 \times 64 \times 4 = 1{,}536$\,KB per CPU.

\textbf{Hot-update advantage.}
Because the LUT resides in a BPF map rather than being compiled into
the program text, model updates require only
\texttt{bpf\_map\_update\_elem()} calls from user space---no program
reload, no XDP detach/attach, and zero downtime.
This enables a future online-learning workflow: retrain the model,
recompute the LUT, and push updates to the running datapath atomically.
No prior in-kernel ML system offers this runtime update capability.

\subsection{TernaryShift: Multiply-Free Hidden Layers}\label{sec:ternary}

\textbf{Problem.}
Each hidden-to-hidden layer computes $h \times h = 4{,}096$
multiplications for $h = 64$, constituting the remaining multiply
operations after MapLUT eliminates the input layer.

\textbf{Approach.}
We quantize each weight to one of three values
$\{-\alpha, 0, +\alpha\}$ and encode the ternary assignment in 2~bits:
\texttt{00}\,=\,0, \texttt{01}\,=\,$+\alpha$,
\texttt{10}\,=\,$-\alpha$.
Sixteen ternary weights are packed into a single \texttt{uint32} word.
The scaling factor $\alpha$ is computed as
$\alpha = \text{mean}(|w| \mid w \neq 0)$ over the layer's trained
weights, and
$\text{alpha\_shift} = \text{round}(16 + \log_2 \alpha)$ converts the
final accumulator to the correct fixed-point scale via a single bit
shift.

\textbf{Ternary dot product.}
Listing~\ref{lst:ternary} shows the core computation.
For each packed word, the code extracts the 2-bit code per weight and
performs a conditional addition or subtraction of the corresponding
activation.
Weights with code~0 are skipped entirely, exploiting the ${\sim}$43\%
sparsity observed in the QAT-trained model (45.6\% in Layer~1, 41.8\%
in Layer~2).
After processing all words, a single right shift by
\texttt{alpha\_shift} replaces the multiplication by $\alpha$.

\begin{lstlisting}[caption={Ternary dot product (simplified).
The inner loop processes 16 packed weights per 32-bit word.},label={lst:ternary},float=t]
static __always_inline int32_t
ternary_dot(int32_t *x, uint32_t *packed,
            int n, int alpha_shift) {
  int64_t acc = 0;
  for (int w = 0; w < n/16; w++) {
    uint32_t word = packed[w];
    for (int k = 0; k < 16; k++) {
      uint32_t code = (word >> (k*2)) & 0x3;
      if (code == 1)      // +alpha
        acc += x[w*16 + k];
      else if (code == 2) // -alpha
        acc -= x[w*16 + k];
      // code==0: zero weight, skip
    }
  }
  return (int32_t)(acc >> alpha_shift);
}
\end{lstlisting}

\textbf{Result.}
Each hidden layer uses zero multiply instructions.
The packed ternary representation compresses the weight matrix from
$h^2 \times 4 = 16$\,KB to $h^2 / 16 \times 4 = 1$\,KB for $h = 64$.
Combined with biases, the full ternary model parameters occupy only
1.8\,KB---a $3.1\times$ compression over the 5.6\,KB int32 baseline.

\subsection{Quantization-Aware Training}\label{sec:qat}

\textbf{The failure of post-hoc quantization.}
Applying ternary quantization to weights trained with standard
cross-entropy produces catastrophic results on minority classes.
On CIC-IDS-2017, PortScan comprises merely 0.08\% of the test data and
BruteForce only 0.55\%; both rely on precise weight values to
distinguish rare attack patterns from the overwhelming benign majority
(85.5\%).
Post-hoc ternarization of the original float model reduces PortScan~F1
from 0.597 to 0.020 and BruteForce~F1 from 0.552 to 0.039
(Table~\ref{tab:ablation})---effectively rendering the model blind to
these attack classes.
The original model's weights are optimized for continuous arithmetic;
coarsely discretizing them destroys the fine-grained distinctions
required for rare-class separation.

\textbf{Two-phase QAT with STE.}
To overcome this, we train a \emph{separate} model specifically for
ternary deployment using a two-phase procedure.

\emph{Phase~1: Float warmup} (30~epochs).
The model trains with focal loss~\cite{lin2017focal} ($\gamma = 3.0$)
and class-balanced oversampling (100K samples per class) to establish
strong minority-class feature representations before quantization
begins.
Batch size is 256; learning rate is 0.01.

\emph{Phase~2: QAT with STE} (90~epochs).
In every forward pass, all hidden-layer weights are quantized to ternary
values $\{-\alpha, 0, +\alpha\}$ using a threshold
$\delta = 0.7 \times \text{mean}(|W|)$:
\begin{equation}\label{eq:ternary}
\hat{w}_{ij} = \begin{cases}
+\alpha & \text{if } w_{ij} > \delta \\
-\alpha & \text{if } w_{ij} < -\delta \\
0       & \text{otherwise}
\end{cases}
\end{equation}
The forward pass uses the ternary weights $\hat{W}$, but the backward
pass computes gradients with respect to the \emph{float shadow
weights}~$W$ via the Straight-Through Estimator~\cite{bengio2013ste,
jacob2018quant}:
\begin{equation}\label{eq:ste}
\frac{\partial \mathcal{L}}{\partial W} \approx
\frac{\partial \mathcal{L}}{\partial \hat{W}}
\end{equation}
This allows the optimizer to continue updating continuous weight
parameters while the model learns representations that are robust to
ternary discretization.
Weight clipping at $|W| > 2.0$ prevents gradient explosion.

\textbf{Why QAT recovers minority classes.}
The critical mechanism is twofold.
First, STE allows gradients from minority-class misclassifications to
propagate through the ternary quantization boundary, enabling the model
to \emph{jointly} optimize which weights remain nonzero and the
magnitude $\alpha$, such that the ternary assignment maximally preserves
classification power for all classes.
Second, focal loss ($\gamma = 3.0$) amplifies the gradient contribution
of hard, misclassified examples---which are disproportionately from
minority classes---forcing the model to allocate its limited ternary
representational capacity toward rare attack patterns.
The result is that QAT-trained ternary weights are specifically
structured for the imbalanced CIC-IDS-2017 distribution, achieving
PortScan~F1 of 0.635---\emph{exceeding} the float baseline's 0.597.

\textbf{Important caveat.}
The QAT model is optimized exclusively for ternary inference.
Its float shadow weights do not constitute a meaningful standalone
classifier (${\sim}$35\% accuracy), because the training objective
drives weight configurations toward ternary-friendly representations
rather than continuous optima.
All QAT results use ternary-quantized weights from the QAT model; all
float/int32 results use the separately trained original model.

\subsection{EarlyExit}\label{sec:earlyexit}

\sys also includes an experimental confidence-gated early-exit
mechanism: a lightweight exit head after Layer~0 that skips the
remaining tail-call stages when the top-2 logit margin exceeds a
threshold.
In principle, this could substantially reduce average latency for
``easy'' traffic (93.1\% of packets trigger early exit).
However, the current implementation drops accuracy to 74.88\% when
combined with MapLUT, because the exit head operates on coarsely
quantized LUT features and compounds discretization error for ambiguous
samples.
We consider EarlyExit a \emph{limitation} of the current system and
leave its improvement---particularly class-aware confidence
thresholds---to future work.
All main results are reported \emph{without} EarlyExit.

\section{Overflow Safety Analysis}\label{sec:overflow}
% ============================================================

A potential concern with the enlargement method is whether the
\texttt{int64} accumulator in Eq.~\eqref{eq:enlarge} can overflow
during quantized inference.
Zhang~\etal~\cite{zhang2024dsn} chose $s = 2^{16}$ empirically but
provided no formal guarantee.
We address this with a worst-case analysis.

\begin{theorem}[Overflow Safety Bound]\label{thm:overflow}
For an MLP $[d_\textup{in}, h, h, d_\textup{out}]$ with scale
$s = 2^b$, let $C_k$ denote the architecture-dependent accumulator
constant at layer~$k$, defined as
$C_k = n_k \cdot W_{\max,k} \cdot X_{\max,k}$ where $n_k$ is the
fan-in, $W_{\max,k}$ and $X_{\max,k}$ are the maximum absolute weight
and activation, respectively.
The \texttt{int64} accumulator does not overflow if and only if:
\begin{equation}\label{eq:safe}
b \;\le\; \left\lfloor \frac{63 -
\lceil\log_2 C_k\rceil}{2} \right\rfloor \quad \forall\; k
\end{equation}
\end{theorem}

For our architecture with $h = 64$ and empirical
$W_\text{max} = 3.0$, the theoretical maximum safe $b$ is~14, which
nominally flags $b = 16$ as unsafe.
However, the worst-case bound assumes all weights simultaneously attain
their maximum absolute value---an extreme scenario that never occurs in
practice.
Empirical analysis on trained weights and 578{,}444 real test inputs
reveals that the actual peak accumulator reaches only
$3.2 \times 10^{-6}$\% of \texttt{INT64\_MAX}, providing a safety
margin of $5{,}590\times$ (12.4~bits of headroom).
This confirms $b = 16$ is safe in practice and establishes a
conservative deployment-validation criterion.

In \sys V3, TernaryShift layers sidestep the issue entirely: ternary
dot products over $h = 64$ activations cannot overflow \texttt{int64}
for any practical hidden size.
MapLUT accumulates only pre-scaled \texttt{int32} additions.
The overflow bound remains relevant for V3b (MapLUT with int32 hidden
layers) and for future architectures that retain full int32 weights in
some layers.

\section{Evaluation}\label{sec:eval}
% ============================================================

\subsection{Experimental Setup}\label{sec:setup}

\textbf{Dataset.}
We evaluate on CIC-IDS-2017~\cite{sharafaldin2018cicids}, a standard
network intrusion detection benchmark containing 2.31M labeled flows
across 4~classes: BENIGN (85.5\%), DDoS (13.9\%), PortScan (0.08\%),
and BruteForce (0.55\%).
We use a 75/25 train/test split, yielding 578{,}444 test samples, with
100K/class oversampling during training.
Twelve features---all extractable from packet headers in the XDP
hook---are used as inputs.

\textbf{Environment.}
All experiments run on Ubuntu~22.04.5 with kernel~5.10.134,
Clang~14, and libbpf~0.5.0.
Models are trained with a NumPy-only MLP implementation (no PyTorch or
scikit-learn dependencies).
The network architecture is $[12, 64, 64, 4]$ with quantization scale
$s = 2^{16}$.

\textbf{Models.}
As described in \S\ref{sec:qat}, we train two models:
the \emph{original model} (cross-entropy, 100~epochs) for Float/Int32
baselines and all post-hoc quantization variants, and the \emph{QAT
model} (30-epoch warmup + 90-epoch STE training with focal loss,
$\gamma = 3.0$) for all ternary-QAT configurations.
Throughout this section, we clearly indicate which model produces each
result.

\subsection{Zero-Multiply Verification}\label{sec:zeromul}

Table~\ref{tab:headline} presents the BPF instruction counts across
four versions of \sys, obtained by disassembling the compiled BPF object
files with \texttt{llvm-objdump} and counting multiply instructions
(\texttt{mul}, \texttt{mul32}).
The baseline V0 uses 804~instructions with 15~multiplications---all from
matrix--vector products.
V2 adds ternary hidden layers and the exit mechanism but retains the
standard input-layer computation, resulting in 1{,}437~instructions with
16~multiplications (one additional multiply from the exit-head logic).
V3 replaces the input layer with MapLUT, achieving 1{,}424 total
instructions with \emph{zero} multiplications---the main result of this
paper.
V3b serves as a hybrid reference: MapLUT for the input layer with
standard int32 hidden layers, yielding 1{,}016~instructions with
5~residual multiplies.

The increase in total instruction count from V0 (804) to V3 (1{,}424)
reflects the overhead of the MapLUT bin-quantization logic and the
ternary unpacking loop, but importantly, \emph{none} of these additional
instructions are multiplications.

\textbf{Lossless int32 quantization.}
The int32 fixed-point baseline perfectly preserves float accuracy
(97.64\%) and macro~F1 (0.770), confirming that the enlargement method
at $b = 16$ introduces no measurable loss.
This is the foundation upon which \sys builds: we replace the multiply
operations within this lossless framework with multiply-free operators.

\textbf{Post-hoc ternary quantization fails on minority classes.}
Directly ternarizing the original model's weights preserves overall
accuracy at 95.02\%---only a 2.6\,pp drop---because BENIGN and DDoS,
which together constitute 99.4\% of the test set, remain largely
correct.
However, PortScan~F1 collapses from 0.597 to 0.020 ($30\times$
degradation) and BruteForce~F1 from 0.552 to 0.039 ($14\times$).
The coarse ternary discretization destroys the precise weight values
that distinguish these rare attack patterns from the overwhelming
benign majority.
Adding MapLUT to post-hoc ternary (row~5) further degrades PortScan to
0.013, confirming that input-layer quantization and weight ternarization
compound errors when both are applied naively.

\textbf{QAT is the breakthrough.}
The QAT model with ternary weights achieves 96.38\% accuracy and
macro~F1 of 0.742---a $1.54\times$ improvement in macro~F1 over
post-hoc ternary (0.482).
The most striking result is PortScan~F1: 0.635 under Ternary-QAT,
which \emph{exceeds} the float baseline's 0.597 by 6.4\%.
This counterintuitive improvement arises because QAT with focal loss
($\gamma = 3.0$) forces the model to allocate its limited ternary
capacity toward correctly classifying hard, underrepresented cases---an
implicit form of capacity prioritization that continuous-valued weights
do not require.

\textbf{Full zero-multiply pipeline.}
MapLUT\,+\,Ternary-QAT at 512~bins achieves 95.78\% accuracy with
macro~F1 of 0.721 and \emph{zero} BPF multiply instructions.
The accuracy gap from the int32 baseline is 1.86\,pp---a modest cost
for complete multiplication elimination.
Notably, PortScan~F1 remains at 0.629, still exceeding the float
baseline, demonstrating that the accuracy cost primarily affects
majority-class margins rather than minority-class detection.

\subsection{Design-Space Exploration}\label{sec:explore}

To validate our architectural choices, we conducted a systematic
48-configuration sweep over the design space: hidden size
$h \in \{8, 16, 32, 64\}$, number of hidden layers
$L \in \{1, 2, 3\}$, and scale bits $b \in \{8, 12, 16, 20\}$.
All 48~configurations completed successfully.

\begin{figure}[t]
\centering
\includegraphics[width=\columnwidth]{figures/fig_scale_bits_accuracy.pdf}
\caption{Int32 accuracy vs.\ scale bits across hidden sizes.
All architectures achieve near-lossless quantization at $b \ge 16$,
confirming that $s = 2^{16}$ is the minimum safe scale factor.}
\label{fig:scalebits}
\end{figure}

Figure~\ref{fig:scalebits} plots int32 accuracy against scale bits
across all hidden sizes.
A clear phase transition is visible: at $b = 8$, quantization error is
severe (accuracy drops 22--31\,pp); at $b = 12$, the gap narrows to
9--17\,pp; and at $b \ge 16$, all architectures achieve near-lossless
quantization (accuracy delta $< 0.05$\,pp).
This confirms that $s = 2^{16}$ is the minimum safe scale factor and
validates our choice of $b = 16$ for \sys.

The Pareto-optimal configuration is $h = 16$, $L = 2$, $b = 16$:
99.62\% accuracy with only 754~BPF instructions and 1.8\,KB model
memory---a useful reference for deployments with tighter instruction
budgets than the 1{,}424 instructions of our $h = 64$ configuration.
All configurations with $b \ge 16$ are overflow-safe per
Theorem~\ref{thm:overflow} ($b_\text{max,safe} \ge 26$ for all tested
architectures).

\section{Related Work}\label{sec:related}
% ============================================================

\textbf{In-kernel ML via eBPF.}
The growing interest in eBPF-based network security has produced several
in-kernel ML systems.
Bachl~\etal~\cite{bachl2021dt} pioneered eBPF-based intrusion detection
with decision trees.
Hara~\etal~\cite{hara2023nn} demonstrated int8 neural inference in
eBPF with significant latency reductions.
Zhang~\etal~\cite{zhang2024dsn} introduced the enlargement method with
tail-call chains---the closest antecedent to \sys, but limited to binary
classification and reliant on multiplications.
Osaki~\etal~\cite{osaki2024} proposed dynamic fixed-point quantization
for eBPF anomaly detection.
Chen~\etal~\cite{chen2025kd} combined knowledge distillation with
decision trees to reduce model complexity.
Zhu~\etal~\cite{o2c2024} developed a compiler framework for offloading
ML models to SmartNICs via eBPF.
All of these systems use multiply instructions; \sys is the first to
achieve zero-multiply neural inference in eBPF.

\textbf{Multiply-free neural networks.}
AdderNet~\cite{chen2020addernet} replaces convolution multiplications
with $\ell_1$-distance computations, achieving competitive ImageNet
accuracy.
DeepShift~\cite{elhoushi2021deepshift} represents weights as powers of
two, converting multiplications to bit shifts.
ShiftAddNet~\cite{you2020shiftaddnet} combines shift and addition
layers in a hardware-inspired architecture.
LUT-NN~\cite{park2024lutnn} uses centroid learning with lookup tables
for efficient mobile inference.
These works target GPU/ASIC environments with float support;
\sys adapts the LUT and ternary concepts to the unique constraints of
eBPF's integer-only, verifier-checked, stack-limited execution model.

\textbf{Binary and ternary neural networks.}
XNOR-Net~\cite{rastegari2016xnor} binarizes both weights and
activations, replacing multiply-accumulate with XNOR~+~popcount.
TWN~\cite{li2016twn} introduced ternary weight networks with symmetric
thresholding.
\sys's TernaryShift builds on TWN but replaces the final scaling
multiplication with a bit shift calibrated to the ternary magnitude,
ensuring full eBPF compatibility.

\textbf{Quantization-aware training.}
Jacob~\etal~\cite{jacob2018quant} showed that training with simulated
quantization in the forward pass and STE in the backward pass yields
integer-arithmetic-only models with minimal accuracy loss.
Bengio~\etal~\cite{bengio2013ste} provided the theoretical foundation
for straight-through gradient estimation.
\sys applies QAT to \emph{ternary} quantization in the setting of
severely imbalanced security data, where we demonstrate that post-hoc
quantization fails catastrophically and QAT with focal loss is
essential for minority-class preservation.

\section{Conclusion}\label{sec:conclusion}
% ============================================================

We presented \sys, the first eBPF/XDP neural inference engine that
eliminates all multiply instructions.
Through two complementary operators---MapLUT for the input layer and
TernaryShift for hidden layers---combined with quantization-aware
training via straight-through estimation, \sys achieves 95.78\% accuracy
on CIC-IDS-2017 with 1{,}424 BPF instructions, zero of which are
multiplications.

The QAT procedure is the critical enabler: post-hoc ternary quantization
catastrophically collapses minority-class detection (PortScan~F1 drops
to 0.020), while QAT with focal loss not only recovers but \emph{surpasses}
the float baseline (PortScan~F1 = 0.629 vs.\ 0.597).
A 48-configuration design-space exploration confirms that our
architectural and quantization choices are well-calibrated, with
$s = 2^{16}$ providing near-lossless int32 inference across all tested
architectures.

\textbf{Limitations.}
The accuracy gap between zero-multiply (95.78\%) and full int32
(97.64\%) is non-trivial at 1.86\,pp.
The 512-bin LUT requires 1.5\,MB per CPU, which may be prohibitive on
resource-constrained embedded devices.
EarlyExit in its current form degrades accuracy to 74.88\% and is not
suitable for deployment.

\textbf{Future work.}
We plan to
(1)~develop class-aware EarlyExit confidence thresholds to safely
bypass computation for easy traffic;
(2)~conduct live XDP deployment with \texttt{tcpreplay} to measure
end-to-end throughput and per-packet latency;
(3)~enable online model retraining with hot LUT updates for concept-drift
adaptation; and
(4)~explore binary (1-bit) weight quantization for further compression.
